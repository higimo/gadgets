## Синхронизация всех репозиториев

Для MacOS синхронизация всех репозиториев в одной директории.

## Польза

Репозитории всегда актуальны: твои коммиты выгружены, чужие загружены.

Чтобы не пулиться перед пушем. Обнаруживать конфликты заранее.

## Требования

Предполагается, что все репозитории тусуются в одной директории

## Использование

```sh
sh git-sync.sh
```

## Исполнение по расписанию

1. Сделать файл исполняемым

```sh
chmod +x git-sync.sh
```

2. Получить полный путь до файла, его надо скопировать

```sh
realpath git-sync.sh
```

3. Начать редактировать крон (внимание, будет VIM)
```sh
crontab -e
```

4. Включить `insert mode`, прожать в английской раскладке `i`


5. Вставить (`Cmd` + `v`) что-то вроде:
```cron
0 8-20/4 * * * /Users/higimo/scripts/git-sync.sh
```

Крайне важно указывать полный путь до файла, cron тупенький

8—20/4 — это запускает скрипт с 8 утра до 20 часов, каждые 4 часа. А именно 8:00, 12:00, 16:00, 20:00.

6. Выйти из `insert mode`, прожать `Esc`

7. Просто набрать `:qw`, затем `Enter`. Что вводите видно в самом низу консоли. `w` — write, `q` — quit.

8. Посмотреть результат

```sh
crontab -l
```

9. Если показывает добавленную строку — всё отлично

10. Если никогда не работали с cron — надо предоставить разрешения cron

## Предоставление разрешений cron MacOS

Если никогда не работали с cron, это обязательный шаг, иначе крон не запускать команду.

1. Системные настройки (System Settings)
2. Конфиденциальность и безопасность (Privacy & Security)
3. Доступ к диску (Disk Access)
4. Прожать +
5. Прожать `Cmd` + `Shift` + `G` или просто `Cmd` + `V`, предварительно скопировав путь в буфер обмена `/usr/sbin/cron`
6. Кнопка «Открыть»
7. Убедиться, что тоггл напротив cron включен
